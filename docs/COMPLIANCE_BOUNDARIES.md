# Límites de cumplimiento normativo

## 1. Qué aporta NexaLab

La plataforma incorpora mecanismos técnicos que apoyan auditorías y operación controlada:

- acceso por roles;
- segregación por laboratorio;
- configuración versionada;
- trazabilidad de muestra;
- historial de cambios;
- inventario por movimientos;
- lotes y vencimientos;
- equipos, planes y certificados;
- métodos y especificaciones versionadas;
- OOS, OOT y CAPA;
- documentos controlados;
- firmas electrónicas;
- respaldos documentables;
- matriz de evidencia.

## 2. Qué no puede resolver únicamente el software

El software no reemplaza:

- acreditación formal;
- revisión legal;
- procedimientos internos;
- capacitación real;
- aprobación de métodos;
- validación de equipos;
- evaluación de proveedores;
- auditorías internas y externas;
- política de retención;
- pruebas de restauración;
- validación computarizada del sistema;
- aprobación de la organización.

## 3. Marcos considerados

### ISO/IEC 17025

La solución incluye estructuras para competencia, equipos, trazabilidad metrológica, métodos, muestras, registros técnicos, reportes, trabajo no conforme, datos, mejora y evidencia.

### ISO 15189

El perfil clínico incluye bases para pacientes, órdenes, identificación, recepción, rechazo, resultados y liberación. Antes de operar clínicamente deben completarse valores críticos, privacidad, remisiones, POCT e integraciones según el alcance del laboratorio.

### BPM y BPL

La plataforma apoya control documental, integridad de datos, capacitación, equipos, reactivos, desviaciones, CAPA, historial y respaldos.

### 21 CFR Part 11

La migración incluye firma vinculada, reautenticación y audit trail append-only. Antes de afirmar cumplimiento debe completarse validación formal, procedimientos, MFA según riesgo, retención, controles operativos y revisión jurídica.

### ISO/IEC 27001

La base incorpora roles, sesiones y segregación lógica. Antes de datos sensibles se requiere gestión de secretos, monitoreo, respuesta a incidentes, respaldo probado, análisis de riesgos y endurecimiento de infraestructura.

### ISO/IEC 25010 e ISO 9001

El código organiza responsabilidades por módulos, utiliza TypeScript, validaciones Zod y documentación de aceptación. El equipo debe mantener pruebas, control de cambios, revisión de código e indicadores.

## 4. Regla comercial recomendada

Promocionar el producto como:

> Plataforma LIMS configurable con controles técnicos para apoyar trazabilidad, calidad y cumplimiento.

Evitar afirmar:

> El laboratorio queda certificado automáticamente al instalar NexaLab.
